Where I started

When I first saw the challenge and saw the grid layout, my mind went to the typical grid layout frameworks like Bootstrap, Bulma, or just plain CSS Grid.

The Challenge

Thinking about it deeper I realized that even if the grid layouts worked fine with those, the boxes when expanded would need to have the other boxes adjust relative to it.

The Decision

I decided to use a library called Masonry.js. "It works by placing elements in optimal position based on available vertical space, sort of like a mason fitting stones in a wall." Perfect.

What next?

The next challenge wasn't so much the resizing of the blocks when clicked (just toggling a class for that), but it was dealing with showing and hiding the text when expanded or collapsed.

Where I started

My first thought was to do the same thing as when expanding the boxes and to just toggle a class called hidden, that would set display from none to block. But this came with some roadblocks

The Challenge

Now that the text is displayed and because it is a child element of the box, clicking certain spots would close the box as expected, but clicking the text would not.

The Decision

Now, we could probably add another function to do listen for a click on the child element and call the same functionality, but adding the style pointer-events: none; seemed to do the trick.

As far as user interactions go, I decided to allow the user to mantain the expanded state on a box when clicking a seperate grid item. The way the boxes resize they take up 1/2 of the grid, so if every box were to be expanded it would be two boxes side by side 50/50 for every "row" unless on mobile. My reasoning was that if a user was looking to compare information in one grid item to another, it seems like it would be rather frustrating as a user to have to re-expand a box to get the info they were looking for in a box they were just looking at.

Well it seems that would conclude my thought process for this code challenge, I hope it gave you enough insight into the decisions I made along the way. Thanks for checking it out!
